## 신모델이 아니라 '마감일'이 뉴스가 된 날: 딥시크 레거시 API의 종료 시각

오늘 AI 모델 소식의 중심에는 새 모델 공개가 아니라 한 개의 시각(時刻)이 있습니다. 한국 시각 7월 25일 새벽 0시 59분(UTC 7월 24일 15:59)을 기점으로 딥시크의 구형 API 별칭이 완전히 접근 불가 상태가 되기 때문입니다. 동시에 '오늘 V4 정식판이 나온다'는 관측이 여러 매체에서 반복됐는데, 공식 문서를 확인하면 사실관계가 다릅니다.

### 딥시크, 오늘 15:59(UTC) `deepseek-chat`·`deepseek-reasoner` 별칭 완전 폐기

딥시크는 공식 API 문서 공지에서 "`deepseek-chat` 및 `deepseek-reasoner`는 2026년 7월 24일 15:59(UTC) 이후 완전히 폐기되어 접근할 수 없다"고 명시했습니다. 두 별칭은 현재 신형 **V4-Flash**로 라우팅되고 있으며, 마감 이후에는 호출 코드에서 모델명을 직접 지정하지 않은 서비스가 그대로 끊깁니다.

폐기와 함께 표준이 되는 V4 계열의 제원은 두 갈래입니다. **DeepSeek-V4-Pro**는 총 파라미터 1조 6,000억(1.6T)에 활성 파라미터 490억(49B) 규모의 MoE 구성이고, **DeepSeek-V4-Flash**는 총 2,840억(284B)·활성 130억(13B)으로 경량화됐습니다. 컨텍스트 길이는 "모든 공식 딥시크 서비스에서 100만(1M) 토큰이 기본값"으로 통일됐고, OpenAI ChatCompletions와 Anthropic API 양쪽 호출 규격을 함께 지원합니다. 활성 파라미터가 총 파라미터의 3% 남짓(49B/1.6T)이라는 점이 핵심인데, 이는 프런티어급 용량을 유지하면서 추론 단가를 억제하려는 MoE 설계의 전형입니다. [DeepSeek API Docs](https://api-docs.deepseek.com/news/news260424/)

### '오늘 V4 정식 출시'설의 실체 — 공식 문서에 GA 일정은 없다

7월 24일을 'DeepSeek V4 스테이블 릴리스 일자'로 소개한 요약 기사들이 다수 유통됐지만, 딥시크 공식 문서에 그런 표현은 없습니다. 문서에 적힌 것은 **2026년 4월 24일자 'V4 프리뷰 릴리스' 공지**와 그 안의 별칭 폐기 시한뿐입니다. 즉 오늘 일어나는 일은 신모델 GA가 아니라 **구형 별칭의 사망 선고**이며, 프리뷰 딱지가 정식판으로 바뀌는 일정은 공개된 바 없습니다.

이 구분이 실무에서 중요한 이유는 비용·안정성 판단이 달라지기 때문입니다. GA 전환이라면 SLA·버전 고정 정책이 함께 따라오지만, 프리뷰 상태의 모델로 트래픽이 강제 이관되는 것이라면 응답 특성 변화에 대한 회귀 테스트를 서비스 측이 직접 떠안아야 합니다. 딥시크는 V4 계열이 클로드 코드, 오픈클로(OpenClaw), 오픈코드 등 주요 에이전트 도구와 통합된다고 밝혔는데, 별칭에 의존하던 에이전트 파이프라인일수록 오늘 마감의 영향을 먼저 받게 됩니다. [DeepSeek API Docs](https://api-docs.deepseek.com/news/news260424/)
