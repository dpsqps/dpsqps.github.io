## 🛠️ Microsoft 에이전트 Windows 11 통합·General Compute ASIC 클라우드 — 에이전트를 위한 새로운 인프라 레이어

4월 18일 운영 체제와 클라우드 인프라라는 두 영역에서 AI 에이전트 전용 기반이 동시에 발표됐습니다. 전 세계 15억+ Windows 사용자와 에이전트 전용 ASIC 클라우드 — AI 에이전트가 범용 소프트웨어 스택을 넘어 컴퓨팅의 기저 레이어로 내려가는 신호입니다.

### Microsoft Windows 11 빌드 26200.8313: AI 에이전트 태스크바 통합 공개 롤아웃 준비 — "@" 트리거·MCP 기반 서드파티 에이전트 지원·Windows Agent API

2026년 4월 18일 Microsoft가 Windows 11 빌드 26200.8313을 Release Preview Channel에 배포하며 AI 에이전트 태스크바 통합의 공개 롤아웃을 준비 중임을 공식 확인했습니다. 작동 방식: 사용자가 태스크바에서 "@"를 입력하면 PC에 설치된 에이전트(예: Microsoft 365 Researcher)가 즉시 활성화되어 OneDrive·M365 파일에 접근해 멀티스텝 리서치 작업을 자율 실행합니다. 기술 기반: MCP(모델 컨텍스트 프로토콜)로 구현되어 서드파티 개발자가 Windows.UI.Shell.Tasks API를 통해 자사 앱을 Windows 에이전트 API에 연결할 수 있습니다. 사용자 제어: 완전히 선택적 기능으로 강제 사용·알림 없음. Microsoft 365 구독이 Researcher 기능 사용에 필요하며, 현재는 Microsoft 365만 태스크바 에이전트를 지원하지만 서드파티 개발자의 채택을 장려 중입니다. 중요한 이유: 전 세계 15억 명 이상이 사용하는 Windows 운영 체제에 에이전트가 OS UI의 기본 구성요소로 통합되는 첫 공식 단계입니다. 에이전트가 브라우저 확장이나 앱 형태가 아닌 운영 체제 레이어에서 실행된다는 것은 배포 방식과 사용 빈도 모두에서 질적 전환점입니다. ### General Compute ASIC-First 추론 클라우드: GPU 없는 에이전트 전용 인프라 — GA 2026년 5월 15일·프리필-디코드 독립 스케일링·수력 발전 운영

2026년 4월 18일 캘리포니아 기반 스타트업 General Compute Inc.(공동창업자: Jason Goodison·Finn Puklowski)가 에이전트 워크로드에 특화된 ASIC 기반 추론 클라우드 플랫폼을 발표하고 일반 공개(GA)를 2026년 5월 15일로 확정했습니다. 핵심 기술 차별점 3가지: (1) **ASIC 기반** — 범용 GPU 대신 목적 맞춤형 AI 가속기를 사용해 에이전트의 고빈도 LLM 호출을 최적화합니다. (2) **프리필-디코드 분리 스케일링** — LLM 추론의 두 단계(프리필·디코드)를 물리적으로 분리해 각 단계의 수요에 맞게 독립 스케일링합니다. (3) **에이전트 자율 프로비저닝** — 에이전트가 직접 API 키를 발급하고 컴퓨팅을 자율로 프로비저닝하는 방식을 네이티브로 지원합니다. 인프라: 수력 발전 데이터센터, 공랭식 가속기, GPU 클러스터 대비 낮은 전력 밀도 운영. 런치 시 다양한 오픈소스 LLM 패밀리 및 사용자 자체 모델 배포를 지원합니다. CTO 제이슨 구디슨은 "지난 20년은 개발자를 위해 빌딩했다. 앞으로 20년은 에이전트를 위해 빌딩할 것"이라고 선언했습니다.
