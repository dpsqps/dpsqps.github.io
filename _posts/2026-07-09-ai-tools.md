## 에이전트를 '한 번에 여러 개' 돌리는 도구들의 진화

오늘의 도구 소식은 하나의 방향을 가리킨다 — 단일 세션·단일 작업에서 벗어나 **여러 에이전트를 병렬로 실행·추적**하는 워크플로다. 편집기와 데이터 파이프라인 양쪽에서 같은 흐름이 나타났다.

### VS Code 1.128, Claude 에이전트 '멀티 챗'과 전용 Agents 창 도입

마이크로소프트가 7월 8일 **VS Code 1.128**을 배포하며 릴리스 노트 맨 앞에 Claude 에이전트 세션용 **멀티 챗(multi-chat)** 기능을 내세웠다. 하나의 세션 안에 연관된 여러 챗을 담아 서로 다른 접근을 **나란히 비교**하고, 이전 턴에서 분기(branch)해 작업을 병렬로 굴릴 수 있다. 여기에 에이전트 우선 워크플로를 위한 **전용 Agents 창**이 추가돼, 여러 프로젝트에 걸친 세션을 한 화면에서 실행·추적한다. '탭 자동완성 → 페어 프로그래밍 → 병렬 에이전트 팀'으로 이어지는 코딩 도구의 진화에서, 편집기가 이제 여러 에이전트의 관제탑 역할을 맡기 시작한 것이다. [Visual Studio Magazine](https://visualstudiomagazine.com/articles/2026/07/08/claude-ai-gets-yet-another-boost-in-vs-code-1-128.aspx)

### Airbyte Agents, OpenAI 앱 마켓 입점 + 'CLI·쓰기 권한'으로 실행형 전환

데이터 통합 업체 Airbyte도 7월 8일 **Airbyte Agents 플랫폼** 대규모 업데이트를 발표했다. 핵심은 세 가지다 — ①**OpenAI 앱 마켓플레이스 입점**으로 챗 인터페이스에서 바로 호출 가능, ②단일 바이너리 **Airbyte Agent CLI** 공개로 터미널·스크립트·CI/CD 파이프라인에 에이전트 직접 삽입, ③**Salesforce·HubSpot 쓰기(write) 권한** 추가. 특히 세 번째가 의미 있다. 지금까지 엔터프라이즈 에이전트는 데이터를 '읽어 오는' 데 그쳤지만, 이제 CRM에 값을 **직접 기록하고 액션을 실행**한다. 조회형에서 실행형으로 넘어가는 전환점으로, 에이전트가 실제 업무 시스템을 건드리기 시작했음을 보여준다. [BigDATAwire](https://www.hpcwire.com/bigdatawire/this-just-in/airbyte-enhances-agents-platform-with-openai-marketplace-launch-and-new-automation-tools/)
