## GitHub Copilot, 기업용 기본 코딩 모델을 'Codex' 계열로 통일

오늘 AI 개발 도구 소식의 중심에는 GitHub Copilot이 있습니다. 수백만 기업 개발자가 매일 쓰는 코딩 어시스턴트의 디폴트 모델이 조용히, 그러나 의미 있게 바뀌었습니다.

### GitHub, Copilot Business·Enterprise 기본 모델을 GPT-5.3-Codex로 전환

GitHub가 공식 체인지로그(2026-05-17)를 통해 Copilot Business 및 Copilot Enterprise 조직의 기본(base) 모델을 기존 GPT-4.1에서 GPT-5.3-Codex로 교체했다고 발표했습니다. 모델 교체 계획 자체는 3월 18일 예고됐고, 이번 5월 17일이 실제 적용일입니다. 주목할 점은 GPT-5.3-Codex가 GitHub의 첫 장기지원(LTS) 모델이라는 사실로, 2027년 2월 4일까지 가용성이 보장됩니다. 기업 입장에서는 자주 바뀌는 모델에 적응하는 부담을 덜고 안정적으로 표준 모델을 운영할 수 있게 된 셈입니다.

구체적인 수치를 보면 전환의 성격이 더 분명해집니다. 새 기본 모델인 GPT-5.3-Codex의 프리미엄 요청 배수는 1x이며, LTS 가용성은 2027년 2월 4일까지 보장됩니다. 기존 GPT-4.1은 당분간 0x 배수로 강제 활성화되지만, 2026년 6월 1일 사용량 기반 과금 도입과 함께 단계적으로 폐기될 예정입니다. 이번 변경은 Pro·Pro+·Free 요금제에는 적용되지 않고, Business·Enterprise 조직에만 해당됩니다.

이 전환이 중요한 이유는, 수백만 기업·엔터프라이즈 개발자가 쓰는 GitHub Copilot의 디폴트 코딩 모델이 코드 생존율(code survival rate)이 높은 Codex 계열로 통일됐다는 데 있습니다. 단순한 버전 업그레이드를 넘어, '에이전틱 코딩' 모델이 엔터프라이즈 개발의 표준으로 자리잡는 흐름을 보여주는 상징적 전환입니다.
