## 구형 Claude 4가 무대에서 내려오다, API에서 두 모델 종료

오늘 AI 모델 소식의 핵심은 '종료'입니다. Anthropic이 1세대 Claude 4 라인업의 두 모델을 API에서 공식 retire하면서, 구형 모델 ID에 의존해 온 프로덕션 환경에 마이그레이션 시한이 닥쳤습니다.

### Anthropic, 초기 Claude 4 모델(Sonnet 4·Opus 4) API 지원 종료

6월 15일부로 Anthropic Claude API에서 `claude-sonnet-4-20250514`(Claude Sonnet 4)와 `claude-opus-4-20250514`(Claude Opus 4) 두 모델이 retire됐습니다. 이날부터 해당 모델 ID로 API를 호출하면 정상 응답 대신 오류가 반환됩니다. 종료된 모델은 정확히 2종으로, Anthropic이 권장하는 대체 모델은 각각 Claude Sonnet 4.6과 Claude Opus 4.6입니다.

이 변화가 중요한 이유는 영향 범위에 있습니다. 프로덕션에서 구형 Claude 4 모델 ID를 코드에 하드코딩해 둔 자동화·통합 시나리오는 종료 시점에 일제히 깨질 수 있습니다. 즉 단순한 버전 업데이트 안내가 아니라, 개발자들이 사실상 모델 마이그레이션을 강제당한 날인 셈입니다. 모델 ID를 외부 설정으로 분리하지 않고 코드 곳곳에 박아둔 팀일수록 충격이 컸을 것이며, 이번 사례는 LLM을 운영에 올릴 때 모델 버전을 교체 가능한 변수로 관리해야 한다는 교훈을 다시 한번 일깨웁니다.
