## 7월 7일 arXiv: 추론 비용, 언어 격차, 임상 배치 — 실전 배포의 세 병목

7월 8일 arXiv cs.AI 카테고리에는 184편이 공시됐다. 그중 전날 투고된 세 편은 각각 장문맥 메모리, 다국어 신뢰도, 의료 현장 통합이라는 서로 다른 배포 병목을 겨눈다.

### DepthWeave-KV: KV 캐시 메모리 8.3배 절감, 64K 문맥에서 초당 72.8토큰

7월 7일 투고된 **arXiv:2607.06523**은 장문맥 LLM 추론의 최대 비용 요인인 KV 캐시를 토큰·레이어별로 다르게 압축하는 기법을 제시한다. 기존 방식이 모든 레이어와 토큰에 균일한 압축률을 적용한 반면, 이 논문의 '교차 깊이 잔차 분해(cross-depth residual factorization)'는 레이어 간에 저랭크 채널 기저를 공유하면서, 어텐션이 정보 손실에 민감한 지점에는 토큰별 경량 잔차를 남긴다. 결과는 **KV 메모리 8.3배 감소**와 **64K 문맥에서 초당 72.8토큰** 처리이며, LongBench·Needle-in-a-Haystack·L-Eval에서 풀 캐시에 근접한 성능을 유지했다. 기저 조회·잔차 역양자화·어텐션 투영을 하나의 연산으로 묶은 fused CUDA 구현으로 디코딩 시 메모리 트래픽까지 줄였다. [arXiv:2607.06523](https://arxiv.org/abs/2607.06523)

### 다국어 불확실성 추정: "영어로 추론시키면 저자원 언어 격차가 사라진다"

같은 날 투고된 **arXiv:2607.06327**은 고·중·저자원 **22개 언어**를 대상으로 **9가지 불확실성 추정(UE) 기법**을 모델 크기·아키텍처별로 비교했다. 핵심 발견은 세 가지다. ①질문이 저자원 언어여도 모델이 **영어로 근거를 생성**하게 하면 불확실성 추정 품질이 크게 개선된다 — 즉 병목은 이해가 아니라 생성이다. ②"영어로 추론하도록 프롬프팅하면 저자원-고자원 언어 간 UE 성능 격차가 닫힌다"는 것으로, 입력 언어의 자원량보다 **출력 언어**가 지배적 변수임을 뜻한다. ③작은 모델은 확률 기반 오픈박스 기법이, 큰 모델은 자기 언어화(self-verbalized) 클로즈드박스 기법이 유리하다. 평가 편향을 줄이기 위해 LLM 심판이나 임베딩 점수를 배제한 설계다. [arXiv:2607.06327](https://arxiv.org/abs/2607.06327)

### LCA: 모델 교체와 임상 워크플로를 분리하는 종양학 오케스트레이션 프레임워크

**arXiv:2607.06531**(7월 7일 투고)은 종양학 임상 의사결정 지원에서 **AI 모델 선택과 임상 데이터 처리를 구조적으로 분리**하는 'Large Cancer Assistant(LCA)' 프레임워크를 제안한다. 단일 시스템에 추론 로직을 내장하는 대신, 멀티모달 데이터 수집·라우팅을 AI 실행부와 떼어냈다. 기하 딥러닝으로 환자 정보를 의학적·구조적 차원에서 표준화하고, 'Cancer Switching Module'이 데이터 흐름을, '표준화 중간 페이로드'가 오케스트레이션과 하위 시스템 사이의 경계를 담당한다. 4개 시나리오 검증에서 정상 운영 시 성능 오버헤드는 무시할 수준이었고, 불완전·이상 데이터를 만났을 때 표적 보완자료 요청(SDR) 생성 **재현율 100%**를 달성했으며, 서로 다른 AI 모델을 교체해도 라우팅 로직은 일관됐다. 논문은 22쪽, 그림 6개, 표 8개, 부록 9개로 Elsevier JBI 형식으로 제출됐다. [arXiv:2607.06531](https://arxiv.org/abs/2607.06531)
